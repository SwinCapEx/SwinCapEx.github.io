EAT40006 Engineering Technology Project B (Eng/CS)

This is a project-based unit in which students develop the skills to generate a computing technology solution to solve one or more industry challenges specifically designed and oriented towards software engineering. This is a 2 semester project.

  • UG_EAT_01 FleetSight: A dive into the insights from vehicle statistics (ATC206)

    Students: Matthew Falcke, Domenic Marulli, Maisha Mehedi, SIYA, Michael Van Der Merwe

    Supervisor: Ali Yavari

    Client: Harindu, Dean, Nadim, Chamod, SkyLedge

    FleetSight is a comprehensive research project that evolved from initially developing machine learning models to determine optimal tyre pressure into an extensive investigation of tyre sensor data and vehicle dynamics. The project conducted a deep dive into tyre sensor performance, analyzing factors affecting signal strength and connection reliability, identifying causes of over and under reporting in sensor readings, and examining consistency patterns in pressure data. A key focus involved developing methods to detect tyre or sensor changes based on data anomalies. Additionally, the project developed a separate vehicle load estimation algorithm that calculates payload distribution.

    Supervisor: Ali Yavari

    Client: Ali Yavari, Swinburne University of Technology

    This project presents Lumetrix, an intelligent UV exposure testbed designed for biological and material research applications. The system integrates environmental control modules managing humidity, temperature, and airflow with a UV-adjustable platform for precise radiation studies. Using flexible 3D-printed components and sensor-driven feedback loops, Lumetrix enables real-time monitoring and automated regulation of experimental conditions.   • UG_EAT_80 Practical Membership Inference Attacks and Efficient Defense in Generative Models (EN103)

    Students: Mabsur Abir, Mohammad Aquib, Anh Ho, Nhat Minh Nguyen, Dat Pham

    Supervisor: Naveed Ali

    Client: Dr. Viet Vo, Swinburne University of Technology

    Knowing which data is used to train large language models is a critical challenge, posing privacy risks to data owners. Recent studies have shown that LLMs are vulnerable to membership inference attacks, which can reveal whether specific data was used to train models such as BERT, GPT, LLaMA, Geok-3, and DeepSeek. However, the practicality of such attacks and defenses remains unclear. For example, some attacks require multiple prompt queries to the server, making them easier to detect, while some defense solutions rely on heuristic thresholds to identify keywords. Furthermore, vision generative models have not been thoroughly investigated under such attacks and defenses. This project will explore new and efficient attack strategies leveraging backdoor attacks, as well as defense techniques using watermarking, malicious query detection, differential privacy, and cryptographic methods.

  • UG_EAT_82 Atomic Source Control for a Quantum Gas Experiment (ATC101)

    Students: Faisal Karim, Aryan Prajapati, Asif Ali Shaik, Niketha Dewmini Thota Hewage, Yannick Wagner, Horatio Yen

    Supervisor: Naveed Ali

    Client: Sascha Hoinka, Swinburne University of Technology

    This project enhances a quantum gas experiment that heats dysprosium metal using Eurotherm PID controllers. Through TCP Modbus communication, we developed a Python Tkinter-based interface to manipulate the controllers, giving researchers greater control and visibility over the experiment. The UI enables real-time comparison of experimental and theoretical temperature data, data export, and scheduling of temperature segments, for example, automatically advancing to the next stage on a specified day and time. By combining hardware control with an intuitive, custom-built interface, the system streamlines experimental workflows and introduces new automation and analysis capabilities for precision temperature regulation in quantum research.
