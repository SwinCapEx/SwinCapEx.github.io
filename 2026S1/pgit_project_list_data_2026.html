COS80029 Technology Application Project

This is a project-based unit in which students work in teams to apply a solution to an industry driven challenge. The project will have a substantial emphasis on technology application. This is the final project for students in Master of IT, Master of IT (Professional Computing), Master of Data Science and Master of Cybersecturity.

  • PG_COS_3 Conquering Inbox Chaos Across Platforms: Office 365 and Zoho Integration for an AI Email Tool (10)

    Students: Senesha Polwaththa Gallage, Sharon Joseph, Srinath Sachithra Mahagama Mahagama Ralalage, Imthath Saabir Mohammed Munabeer

    Supervisor: Rui Zhou

    Client: Jeremy Nagel, Focus Bear

    -- What is BearlyMail? -- BearlyMail is an AI-powered email management tool that helps people (especially those who struggle with email overwhelm) take control of their inbox through intelligent batching, prioritisation, and snoozing. It currently supports Gmail, and we're expanding to support more email providers. -- What You'll Do -- We have a proof-of-concept integration for Office 365 (Outlook/Hotmail) and Zoho Mail, but it hasn't been tested or made production-ready. Your job will be to: Test the existing proof-of-concept integrations and identify what's working and what's broken Fix bugs and handle edge cases across both providers Ensure feature parity with the existing Gmail integration (batching, prioritisation, snoozing, urgent filtering) Write automated tests to verify the integrations work reliably Handle differences in how each email provider's API works (e.g. how labels/folders, read status, and push notifications behave differently across Gmail, Office 365, and Zoho) Document any provider-specific quirks for the team -- AI-Assisted Development -- Like all projects at Focus Bear, you'll be expected to use AI coding assistants (Claude, GitHub Copilot) to accelerate your work ó learning how to effectively prompt, review, and iterate on AI-generated code. === TECH STACK === Microsoft Graph API (Office 365) Zoho Mail API Gmail API (existing reference implementation) AWS (backend infrastructure) NestJS (backend framework) AI coding assistants (Claude, GitHub Copilot) Prior experience with all of these isn't required ó what matters is willingness to learn and problem-solve. === WHY IT MATTERS === Millions of people use Office 365 and Zoho Mail, and email overwhelm isn't limited to Gmail users. By expanding BearlyMail's provider support, you'll be helping make the product accessible to a much wider audience ó including workplaces and organisations that standardise on Microsoft or Zoho.

  • PG_COS_7 Threat Chain Explorer: Visual Analytics Platform for MITRE ATT&CK (10)

    Students: Nafiz Imtiaz, Farhan Monsur, Avadh Vora, Keyur Sandipbhai Patel, Nishandeep Singh, Aniket Solanki

    Supervisor: Yi Tian

    Client: Pei-Wei Tsai, Department of Computing Technologies, Swinburne

    The MITRE ATT&CK framework is a globally recognised knowledge base that categorises adversarial tactics, techniques, and procedures (TTPs) across different stages of the cyber attack lifecycle. It is widely adopted by red teams, blue teams, and threat intelligence analysts to understand attacker behaviour. However, the framework is primarily presented in a matrix format, which limits its ability to represent sequential attack paths, inter-technique relationships, and dynamic adversarial progression. This project aims to design and develop an interactive visual analytics platform that transforms the MITRE ATT&CK framework into structured, navigable attack chains. Instead of viewing techniques in isolation, students will model ATT&CK data as a graph-based structure and enable exploration of adversarial pathways through chain-based visualisation.

  • PG_COS_8 Intelligent Phishing Detection Platform: LLM-Based Analysis and Deployment (9)

    Students: Fereshteh Sokhanguei, Anmol Srivastava, Ritesh Ramesh Gore, Indusara Abeygunawardana, Tharuvi Mathangaweera

    Supervisor: Yi Tian

    Client: Pei-Wei Tsai, Department of Computing Technologies, Swinburne

    With the increasing sophistication of phishing emails, traditional rule-based detection systems struggle to capture subtle linguistic manipulation and context-aware deception. Modern large language models (LLMs) offer new opportunities for semantic understanding and contextual analysis of malicious content. This project approaches phishing detection as a data-driven text analysis and system deployment problem. Students will design and implement an intelligent phishing detection platform powered by large language models, focusing on semantic reasoning, contextual interpretation, and explainable classification.
